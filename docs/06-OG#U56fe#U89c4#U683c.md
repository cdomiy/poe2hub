# OG图规格速览

详细规格文档在 `og-templates/OG_SPEC.md`，这里只做速览索引，避免两处内容重复维护。

## 核心信息

- 尺寸：1200 × 630px，四周留60px安全区
- 输出：构建时预渲染PNG，不用运行时截图方案
- 技术方案：SVG占位符模板 + `@resvg/resvg-js` 栅格化，适配Cloudflare Pages构建流程
  （不用`@vercel/og`，因为依赖Vercel Edge Runtime）

## 四类页面的差异化视觉锚点

| 页面类型 | 视觉锚点 | 强调色 |
|---|---|---|
| Build攻略 | 右上角仿"稀有度宝石"式Tier徽章 | 氧化铜绿 |
| BOSS攻略 | 底部阶段圆点（阶段数量直观可见），不足2阶段自动降级显示推荐等级 | 血锈色 |
| 机制百科 | 左上角"常青更新"标签+当前版本号 | 金色 |
| 补丁速报 | 版本号本身作为超大数字视觉主体 | 血锈色（更饱和） |

## 涉及的文件

```
og-templates/
├── OG_SPEC.md          # 完整规格文档（字段映射、字体处理、兜底逻辑等）
├── build.svg            # Build页模板
├── boss.svg              # Boss页模板
├── mechanics.svg         # 机制页模板
└── patch.svg             # 补丁页模板

scripts/
└── generate-og.mjs      # 批量生成脚本，读取content collections的frontmatter
```

## 接入前需要准备的东西

- 3个字体文件（Cinzel-SemiBold / Inter-Medium / IBMPlexMono-Medium），放进
  `og-templates/fonts/`
- `npm install gray-matter @resvg/resvg-js fast-glob`
- 把 `node scripts/generate-og.mjs` 接入构建流程（建议作为 `astro build` 前的
  `prebuild` 脚本）
