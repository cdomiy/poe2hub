# 内容优先级排期

排期逻辑：结合游戏版本节点（补丁密集期 → 大型活动/发布会 → 正式版上线）安排内容产出顺序，
而不是按内容类型平均分配精力。

## 分阶段排期表

| 阶段 | 重点内容 |
|---|---|
| 第一批（版本活跃期开始） | Tier List、当前联盟机制解读页、新升华/新职业build、Atlas/系统类专题（官方陆续放出的部分，适合追热点） |
| 第二批 | 补全剩余职业build矩阵、Boss攻略补全、常青机制页打磨 |
| 第三批 | 新手区查漏补缺，准备承接下一波新玩家流量 |
| 第四批（大型活动/发布会前后） | 前瞻资讯页、正式版预测/爆料追踪，这段时间news板块流量会爆发 |
| 第五批（正式版上线） | 全站Build页/流程页大改版（大版本平衡改动通常很大），新手区重写 |

## 排期背后的判断依据

1. **补丁活跃期**：机制解读+Build Tier List是流量核心，因为版本还在连续打补丁，
   玩家反复查"这个东西是不是又改了"
2. **护城河风险点**：如果官方推出"游戏内build导入"一类功能并已经在联系头部攻略站做适配，
   意味着单纯"文字build攻略"的护城河会变浅——如果有能力，尽早考虑做可导入的build文件/
   工具类内容，是差异化机会
3. **大版本节点**（发布会+正式版上线）是下一波大流量节点，需要提前规划前瞻类页面骨架，
   提前卡位而不是等流量来了才开始写

## 排期使用方式

- 这份排期不是固定不变的日历，而是"节点驱动"的判断框架——具体日期要跟着实际的版本
  更新计划走
- 每次大版本更新后，应该重新对照这份文档评估一遍：哪些内容属于"下一批"该做的，
  哪些原本排期靠后的内容因为版本改动提前变得紧急
