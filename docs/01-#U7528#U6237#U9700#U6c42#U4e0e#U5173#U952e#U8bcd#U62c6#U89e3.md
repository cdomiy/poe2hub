# 用户需求与关键词拆解

## 一、用户搜索意图的五个大类

PoE2这类深度RPG攻略站的搜索意图大致分五种，决定了内容的"急迫程度"和变现潜力：

| 意图类型 | 特点 | 变现价值 |
|---|---|---|
| 版本追新（Info） | 想第一时间知道"这个版本改了什么" | 流量大但生命周期短，适合做首页/新闻聚合 |
| 决策型（Build怎么选） | "我该玩什么职业/技能" | 长尾流量稳定，AdSense点击率高（用户会反复回来看） |
| 卡关求助（How-to） | 打不过某个Boss/机制不懂 | 精准长尾词，转化率最高，竞争相对小 |
| 效率优化（Min-max） | 已经会玩，想更快更强 | 用户粘性强，适合互动工具（如favor tracking式组件） |
| 数据查询（Reference） | 查数值、掉落、图纸 | 需要结构化数据库，SEO friendly但开发成本高 |

## 二、关键词方向具体拆分

### 1. 新手指南类（Beginner Guide）
- `poe2 beginner guide` / `poe2 how to start`
- `poe2 class guide`（7-8个职业+对应升华）
- `poe2 campaign walkthrough`（分Act，0.5.0对campaign做了大改，旧攻略需重写）
- `poe2 leveling uniques` / `poe2 best leveling skills`
- `poe2 currency guide`（新手最容易卡住的点）

### 2. Build攻略类（最大流量池）
按"职业 × 升华 × 核心机制"三级细分，而不是笼统写"XX build"：
- `[ascendancy] build guide`（如 spirit walker build, martial artist build）
- `[skill gem] build`（如 minion build, bleed build, lightning build）
- `poe2 tier list`（每次平衡补丁后搜索暴增，是引流利器）
- `poe2 best budget build` / `poe2 best league starter`（联盟开荒需求强）
- `poe2 endgame build` vs `poe2 leveling build`（需求不同，要分开写）

### 3. BOSS打法类
- `[boss name] guide`（尤其是版本调整过机制的Boss，搜索量会临时暴增）
- `poe2 pinnacle boss guide`
- `poe2 boss mechanics explained`
- 可结合"打法模拟器"思路做互动组件：选择"我在哪个阶段卡住"→跳转对应说明

### 4. 机制/系统解读类（版本特化，时效性强）
- `[联盟机制名] guide`（如 runes of aldur guide / runeforging guide）
- `poe2 atlas masters explained`
- `[reworked mechanic] rework guide`（如 breach rework guide）
- `poe2 logbook / island exploration guide`

### 5. 装备/制造类
- `poe2 crafting guide`
- `poe2 unique upgrade`
- `poe2 rune guide`

### 6. 资讯/趋势类
- `poe2 patch notes [版本号]`
- `poe2 1.0 release date`
- `exilecon [年份]`
- 这类词短期爆量，适合做"快讯页+内链导流到Build/机制深度页"，避免首页流量白白流失

## 三、优先级建议的判断逻辑

1. 版本活跃期（补丁密集发布阶段）：机制解读+Build Tier List是流量核心，玩家会反复查
   "这个东西是不是又改了"
2. 竞争格局变化：如果官方推出"游戏内build导入"一类功能并与头部攻略站合作，
   意味着纯文字build攻略的护城河会变浅，需要尽早考虑可导入文件/工具类内容做差异化
3. 大版本节点（如资料片发布会、正式版上线）是下一波流量高峰，需要提前规划前瞻类页面骨架卡位
