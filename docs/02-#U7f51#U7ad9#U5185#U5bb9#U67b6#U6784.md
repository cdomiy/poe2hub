# 网站内容架构

## 一、宏观结构

```
首页
├── 版本速报（置顶轮播，链接到深度攻略）
│
├── 新手区 /getting-started/
│   ├── 新手大礼包（选职业、练级、货币入门）
│   └── 剧情流程（分Act，标注版本改版内容）
│
├── Build中心 /builds/
│   ├── Tier List（版本更新即改，SEO权重最高的页面之一）
│   ├── 按升华分类（每个升华下挂3-5个具体build）
│   ├── 按预算分类（开荒/自组/满配）
│   └── [互动工具] Build筛选器/技能连招模拟器
│
├── BOSS攻略 /bosses/
│   ├── 剧情Boss
│   ├── Pinnacle Boss
│   └── [互动] 阶段模拟/机制拆解图
│
├── 机制百科 /mechanics/
│   ├── 本赛季联盟机制专题
│   ├── 常驻系统（图谱、Delirium、Breach等，每次rework都更新同一页保留权重）
│   └── 制造/工艺
│
├── 资讯 /news/
│   └── 补丁解读（每条都反向链接到受影响的Build/机制页）
│
└── 工具 /tools/（技术允许的话，比头部站点更轻量的差异化点）
```

## 二、完整URL结构

```
/                                    首页（版本速报+入口导流）

/getting-started/
  /class-guide/                      职业总览（对比表）
  /class-guide/[class]/              单职业详情
  /ascendancy/[name]/                升华详情
  /campaign-walkthrough/act-[n]/     分幕流程
  /leveling-guide/                   练级路线总览
  /currency-guide/                   货币体系
  /controls-and-basics/              操作/UI说明

/builds/
  /tier-list/                        当前版本T列表（核心页，高频更新）
  /tier-list/[patch]/                历史版本T列表存档（长尾+可信度）
  /[class]/[ascendancy]/[skill]-build/   具体build页
      例：/huntress/spirit-walker/bleed-build/
  /league-starter/                   开荒推荐合集
  /budget-builds/                    低成本向合集
  /endgame-builds/                   满配向合集

/bosses/
  /campaign-bosses/[name]/           剧情Boss
  /pinnacle-bosses/[name]/           终局Boss
  /boss-mechanics-explained/         机制通用解读

/mechanics/                         （常青页，随版本更新同一URL）
  /atlas/                           图谱系统总览
  /atlas/masters/[name]/            Atlas Masters专题
  /league-mechanic/[name]/          本赛季机制深潜
  /delirium/ /breach/ /expedition/  常驻机制
  /crafting/                        制造总览
  /crafting/rune-crafting/          Rune相关
  /unique-upgrade-guide/            低级极品升级endgame机制

/news/
  /patch-notes/[version]/           补丁解读
  /roadmap/                         版本前瞻追踪页

/tools/
  /build-planner/                   （远期，工程量大）
  /tier-list-interactive/           可筛选/排序的tier list
  /boss-mechanic-simulator/         阶段模拟/机制拆解交互
```

## 三、URL设计原则

- Build页固定用 `/[class]/[ascendancy]/[skill]-build/` 三级路径，方便未来批量生成和内链归类
- 机制百科页坚持"同一URL跨版本持续更新"，不为每次改版单独开新页——这是常青页策略的核心
- 补丁速报页走时间序列URL（`/patch-notes/[version]/`），短命流量通过内链导流到常青页承接权重
