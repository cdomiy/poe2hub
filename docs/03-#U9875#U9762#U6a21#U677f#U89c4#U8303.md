# 页面模板规范

每类页面结构固定下来，方便批量生产，也方便SEO和用户扫读。对应的实际实现见
`src/pages/` 下的四个动态路由文件。

## 模板A：Build攻略页

对应实现：`src/pages/builds/[...slug].astro`

```
1. TL;DR摘要框（顶部）：适合人群、优点缺点、预估成本、当前版本强度
2. 技能连招/主动技能列表（图标+简述）
3. 天赋树链接/截图
4. 装备优先级（分：核心必须 / 加成推荐 / 奢侈品）—— 对应 ItemTooltipCard 组件
5. 升华天赋加点顺序
6. 练级路线（该build何时可用、前置技能替代方案）
7. 优缺点/克制关系
8. 常见问题FAQ（3-5条，天然承接长尾搜索词）—— 对应 FaqAccordion 组件
9. 版本更新记录（复用页面而不是重开新页）
```

## 模板B：BOSS攻略页

对应实现：`src/pages/bosses/[...slug].astro`

```
1. 基础信息（推荐等级/抗性/掉落）
2. 阶段拆解（Phase 1/2/3，含关键机制触发条件）
3. 常见死亡原因 & 应对（这是最容易被搜索的部分）
4. 推荐打法（可结合互动模拟器：选择"我在哪个阶段卡住"→跳转对应说明）
5. 该Boss专属掉落/图纸价值（关联到build页做内链）
```

## 模板C：机制百科页（常青页策略）

对应实现：`src/pages/mechanics/[...slug].astro`，配合 `ChangelogTimeline` 组件

不是每次改版都新开一页，而是维护一个"活文档"：

```
1. 当前版本机制说明（顶部标注"最后更新于 X.X.X"）
2. 历史变更时间线（折叠式，SEO友好又不影响可读性）
3. 实战技巧
4. 相关build/资源链接
```

这样做的好处：单页面持续累积外链和权重，而不是每个版本都从0开始。

## 模板D：补丁速报页

对应实现：`src/pages/news/patch-notes/[...slug].astro`

```
1. 一句话总结（"这次改了什么，对你有什么影响"）
2. 分类变更（职业/机制/物品）
3. 【重要】每条变更下方直接内链到受影响的build/机制常青页
   —— 这一步是把"短命流量"转化为"常青页权重"的关键动作
```

## 模板选择判断表

| 内容类型 | 用哪个模板 | 判断依据 |
|---|---|---|
| 具体某个build的完整打法 | 模板A | 有明确的职业+升华+技能组合 |
| 某个具体Boss的打法 | 模板B | 有明确的阶段/机制可拆解 |
| 某个系统性玩法机制的解读 | 模板C | 内容会随版本持续更新，不是一次性的 |
| 某次版本更新的变更汇总 | 模板D | 内容有明确的时间戳和版本号，属于新闻性质 |
